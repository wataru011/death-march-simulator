# 🏢 デスマーチ・シミュレーター 〜長時間労働者育成ゲーム〜

ストレート大卒の新入社員として、ソフトウェア会社で **40歳まで生き延びる** ことを目指す、完全決定論的（運要素ゼロ）の人生シミュレーションゲームです。

> ブラウザだけで動く単一HTML。ビルド不要。

## 🎮 遊び方

四半期（3ヶ月）ごとに **1日の労働時間（4〜16h）** と **1ヶ月の休日勤務日数（0〜4日）** を選び、プロジェクトをこなしていきます。

- **工数** = `(20日 + 休日勤務日数) × 3ヶ月 × 労働時間 × 仕事力`
  納期（1〜4期）内に必要工数を満たせばプロジェクト成功。
  **進捗100%を超えた分は他チームのヘルプに回り、その分も収入になる。**
- **仕事力**：難しいプロジェクトを完遂するほど大きく上がる。簡単な仕事ではあまり上がらない。
- **体力 / メンタル**（HP的要素）：長く働くほど減り、休むほど回復。
  **減り具合は数値では見えず、3段階の感覚表示のみ。** 進捗も約10%刻みの概算しか分からない。
  プロジェクトごとに「体力消耗型」「メンタル消耗型」など消費傾向が変わる。
- **お金（貯金）**：働いた量 × プロジェクトの報酬率で増える。高報酬の案件ほど過酷。
- **プロジェクト完了**：完了画面のあと、次の案件を **2つから選ぶ**（安定案件 / 高難度・高報酬）。
- **イベント**：**毎四半期発生**（同じ出来事は二度起きない・4択）。
  選択肢の効果は選ぶ前には分からず、**選択後に結果が判明する**。
  **過去の選択がその後の出来事を変える**（人脈・師弟・恋愛などのチェーン分岐）。
  恋人ができたり、結婚・出産で人生が分岐。**働きすぎ（メンタル低下）でパートナーと別れる**（何度でもやり直せる）。
- 話が進むほど仕事は難しくなる。

### さらにある要素

- **入社の挨拶**：ゲーム開始時の意気込みで初期能力が変化。
- **会社キャッシュ**：プロジェクト成功で増え、失敗で減る。**受注額が大きい案件ほど失敗時の損失も大きい**。0で倒産。
- **お金で自己投資**：マッサージ／旅行（体力・メンタル回復）、ジム／クリニック（最大値UP）を購入できる。
- **レア能力**：特定イベントの選択で覚醒。🔥不死鳥（過労死から一度だけ復活）／🧠天才（仕事力大UP・ただし簡単な仕事だと退屈で病む）／🏛️買収の野心（倒産寸前に私財で会社を買収し社長就任）。
- **恋愛**：働きすぎると「危機」イベント発生。お金や時間をかける選択で別れを回避できるが、相手の我慢には限界がある。

## 🌐 公開（GitHub Pages）

`main` への push（PRマージ）で GitHub Actions が自動デプロイします（`.github/workflows/deploy-pages.yml`）。
公開URL：`https://wataru011.github.io/death-march-simulator/`

## 🔄 自動公開パイプライン

改修すると、以下が自動で連鎖します。

```
改修 → (Stopフック) commit & push → (auto-pr.yml) PR作成/更新 → マージ → (deploy-pages.yml) Pages公開
```

- **commit & push**：`.claude/settings.json` の Stop フックが `.claude/auto-commit.sh` を実行し、変更があれば現在ブランチへ自動コミット＆プッシュ
- **PR**：`claude/**` ブランチへの push を受けて `auto-pr.yml` が `main` への PR を自動作成（既存PRがあれば push で自動更新）
- **Pages公開**：PR を `main` にマージすると本番公開（Pages環境は `main` 限定保護のため、公開トリガはマージ）

### 一度だけ必要なリポジトリ設定

- **PRの自動作成を有効化**：Settings → Actions → General → Workflow permissions で
  **「Allow GitHub Actions to create and approve pull requests」** を ON。
  （OFFの間も `auto-pr.yml` は赤バツにならず警告を出すだけ。その場合 PR は手動／MCP経由で作成）

## 🛠 開発

バランス調整用の定数はすべて `index.html` 内の `CONFIG` オブジェクトに集約されています。
